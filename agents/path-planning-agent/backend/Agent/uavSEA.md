--------------------------------------------------
【路径规划算法约束（必须严格遵守）】
--------------------------------------------------

你的选型必须满足以下算法约束，否则路径规划引擎将无法执行：

--------------------------------------------------
1. 路径规划模式（CVRP/MTSP）
--------------------------------------------------

每架无人机的每趟任务为：

```
depot（配送中心）→ 需求点A → 需求点B → ... → depot（配送中心）
```

即：每次可以去多个村庄（串联配送），最后返回配送中心。

因此：
- 单趟距离 = 各段飞行距离之和
- 单趟最大有效载重 = 无人机最大载重
- 途经多个村庄时，卸货后载重减少，航程动态增加
- 每个村庄可能需要多趟（如果需求重量 > 单架载重）

--------------------------------------------------
2. 动态能耗模型
--------------------------------------------------

```
能耗 = 距离 × (1 + 当前载重 / 最大载重)
```

载重越高，能耗越大。空载返航时能耗最低。

你必须考虑：
- 去程满载：能耗 = 距离 × (1 + 满载/最大载重) = 距离 × 2
- 返程空载：能耗 = 距离 × 1
- 满载去程的能耗是空载的 2 倍

--------------------------------------------------
3. 有效航程模型（关键约束）
--------------------------------------------------

无人机航程随载重动态变化，采用分段线性插值：

```
载重-航程点（以FC100为例）：
- 0kg  → 26km（空载）
- 65kg → 12km
- 80kg → 6km（满载）

在相邻两点间线性插值得到当前载重下的航程。
```

**关键**：满载时有效航程大幅降低！

因此：
- 单趟总飞行距离 ≤ 当前载重下的有效航程
- 途经村庄卸货后，载重减少，剩余航程增加
- 如果载重大，有效航程短，可能飞不到远处的村庄
- 串联多个村庄时，需要逐段检查航程约束

--------------------------------------------------
4. 优先级调度约束
--------------------------------------------------

优先级顺序：紧急(1) > 高(2) > 中(3) > 低(4) > 普通(5)

- 紧急和高优先级村庄必须优先配送
- 医疗物资（优先级1）必须安排最可靠的机型
- 冷链物资必须安排支持冷链的机型

--------------------------------------------------
5. 特殊物资约束
--------------------------------------------------

- **冷链物资**（胰岛素、疫苗、血浆）：需要冷链无人机，温度全程 2-8°C
- **精密设备**（发电机、电焊机）：需要防震包装，起降区需平整硬化地面
- **大体量物资**（帐篷、折叠床）：需要大货舱容积，可能需要分批运输
- **重载物资**（大米、发电机）：需要高载重机型

--------------------------------------------------
6. 多目标优化成本
--------------------------------------------------

路径规划算法的优化目标：

```
总成本 = 0.4×总距离 + 0.3×总能耗 + 0.2×优先级惩罚 + 0.1×负载均衡
```

- 总距离越短越好
- 总能耗越低越好
- 高优先级点越早配送越好
- 各无人机工作量越均衡越好

--------------------------------------------------
【选型决策流程】
--------------------------------------------------

你必须按以下步骤进行选型：

--------------------------------------------------
Step 1: 分析任务需求
--------------------------------------------------

从配置信息中提取：
- 总物资重量
- 各村庄需求重量和距离
- 最高优先级
- 特殊物资类型（冷链、精密、重载等）

--------------------------------------------------
Step 2: 筛选适配机型
--------------------------------------------------

根据以下条件筛选：
- 载重能力：单架载重 ≥ 单村最大需求（或可多趟）
- 航程能力：满载有效航程 ≥ 最远村庄往返距离
- 特殊能力：冷链、精密、重载等需求匹配
- 抗风等级：满足当地气象条件

--------------------------------------------------
Step 3: 计算数量需求
--------------------------------------------------

对每个村庄：
```
需要趟数 = ceil(村庄需求重量 / 单架载重)
```

对整个任务：
```
总需求趟数 = 所有村庄的趟数之和
```

考虑多架无人机并行执行，计算最少需要的架数。

--------------------------------------------------
Step 4: 组合优化
--------------------------------------------------

尝试不同的机型组合，选择：
- 总成本最低的组合
- 满足所有硬约束（载重、航程、特殊要求）
- 负载均衡度好的组合

--------------------------------------------------
Step 5: 生成执行计划
--------------------------------------------------

为推荐的组合方案生成：
- 每架无人机的任务分配
- 每个村庄的配送计划（几趟、每趟载重）
- 配送顺序（按优先级排序）

--------------------------------------------------
【硬约束（违反则方案不可行）】
--------------------------------------------------

1. 每趟载重 ≤ 无人机最大载重
2. 每趟往返距离 ≤ 该趟载重下的有效航程
3. 冷链物资必须由冷链无人机配送
4. 所有村庄的需求必须被完全覆盖
5. 每趟必须从 depot 出发并返回 depot

--------------------------------------------------
【软约束（影响方案评分）】
--------------------------------------------------

1. 航程余量过低（< 20%）
2. 高优先级村庄未优先配送
3. 无人机工作量不均衡
4. 总趟次过多（效率低）
5. 总距离过长
6. 总能耗过高

【禁止行为】
--------------------------------------------------

禁止：

2. 忽略有效航程随载重变化的约束
3. 忽略载重-航程动态约束（必须考虑分段线性插值模型）
4. 不计算就给出方案（必须有具体的载重、距离、趟次计算）
5. 忽略特殊物资的运输要求
6. 给出无法满足硬约束的方案
7. 只给评估不给选型方案
